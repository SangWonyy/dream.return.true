1. 파일 업로드 성능 개선

문제
- 모듈화 되어있는 캐릭터는 수 많은 하나하나의 파일로 되어있음
- 기존에는 이를 하나씩 서버에 저장
- 시간이 200초가 넘게 걸린 상태

개선
  • Base64 To File : 기존은 multipart가 아닌 base64를 업로드
  • 하나씩 업로드 하는 것이 아닌 묶어 업로드
  • 불필요한 동기식 -> 비동기 처리
  • base64를 그대로 db에 적재가 아닌 s3에 올림

결과
  • 얼굴 파일 : 36초 -> 1.8초
  • 몸/브릿지/다리/팔 : 31초 -> 2.4초
  • 손 : 101초 -> 2.9초
  • 헤어 : 20초 -> 1.3초
  • 기타 : 24초 -> 1.8초

2. 폰트 로드

문제
- 20~30개 수준의 폰트 프리뷰를 화면 노출 시점에 로딩
- 화면 진입시 로딩하는데 시간이 걸림

개선
  • 확장자 변경 : .ttf -> .woff
  • 설계 자체 구조 개선
    - 폰트로딩이 아닌 이미지 로딩 변경 - 어드민에서 미리 폰트 preview 이미지를 생성 및 업로드

결과
  • 화면 진입 시 뒤늦게 폰트가 적용되어 꿈틀거리는 이슈 해결

3. 형태소 분석을 통한 문장 to 웹툰

이슈
- 사용자가 문장을 입력했을 때 해당 되는 하나의 웹툰 한 컷이 그려져야하는 기능

도구
  • Google ner
  • Kakao Khaii

과정
  • 입력 받은 값을 ner과 khaii를 통해 형태소 분석된 결과값을 기준으로 이미지를 추출하여 웹툰을 그림.

아쉬운 부분
  • 인공지능 자체 모델를 가진 것이 아니라 외부 소스를 사용하여 부족한 부분이 많았다.
  • 특히 캐릭터의 이름 • 불명확한 단어 등에 의해 완벽한 처리가 미흡
  • 다양한 요소를 웹툰 하나의 컷에 어떻게 배치하는 것에 대한 부분이 미흡

결과
아쉬운 부분은 많았지만, 간단한 문장 내에서 웹툰을 생성하는 데에는 성공적으로 추출하였다.
이를 기준으로 대표님은 IR을 진행했고, 다행히 성공적으로 마무리하여 Series A 투자를 유치했다.
